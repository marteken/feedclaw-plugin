# Email Newsletter Guidelines

## Structure

- `title` field = subject line (also rendered as H1 in the email)
- `content` field = markdown body rendered to styled HTML by FeedClaw

## Formatting

- Double newline = new paragraph
- `## Heading` for section titles
- `**bold**` for emphasis
- Links: `[text](url)` — rendered as styled anchor tags
- Do NOT use `![](url)` for images in content body — use the `image_url` field instead

## Length and tone

- Ideal: 300–600 words
- End with a clear, specific CTA
- One main idea per newsletter — don't cram multiple topics

## Subject line best practices

- Specific > vague: "3 mistakes killing your AV installs" > "Tips for integrators"
- Ask a question or make a bold claim
- Under 50 characters for mobile preview

## Common mistakes to avoid

- Using the same text as the X post
- Embedding image URLs as markdown `![](url)` — they appear as broken text
- Multiple competing CTAs — pick one
